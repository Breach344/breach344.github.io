FGO Duel(Video Game)

The rules are on the official site. It's still a work in progress but I wanted to show this off. Your pieces on the board are assigned individual classes that get removed and added from the tiles when you move in order to keep track of their locations. Turn phases are done with a number counter. Only certain actions are allowed for each phase(only when the turn counter variable is at certain values). Each card has an individual id to identify its type, power, and game piece. If all cards used in battle have a type in common they create a chain with varying abilities that could last several turns which I did by using variables that tick down counters when turns end. When a unit attempts to land on a space with an enemy class on it then it will start the battle function where it checks for card power, card chains, and the counter variables to see who wins. All functionality is tested. Only data entry, optimizing, and styling remains.

Neural Drive

The canvas element is controlled with Javascript functions in real time to create a nearly impossible game for a human to complete. Next I created a self learning AI to solve it in order to showcase an example of how an AI can doing what humans can't when used properly. You can select "Game" to try playing, "Network" to activate the neural network which you can watch learn to play until it wins, or "Best Brain" to see a brain that's already beaten the game. The AI sees through invisible lines in front of it that return a value based on how early they're interrupted by an object. It learns through a form of reinforcement learning where the car that goes the farthest gets the highest score and it's brain continues to the next game. The AI's brain is a bunch of values for where and if it should turn depending on the values it sees determining surrounding object distances. Each run it tries changing each variable that decides if and where it should turn by a certain percentage and runs a number of simulations to see if any result in a new high score.
